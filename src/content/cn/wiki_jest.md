# Jest / Vitest: 你的代码保镖

> "没有测试的代码，就是待宰的羔羊。"

**Jest** 和 **Vitest** 是前端最流行的测试框架。它们是用来"测试代码的代码"。

---

## 为什么要写测试？

想象一下，你修了一个 Bug，结果把登录功能搞挂了。你不知道，直接发布了。
如果写了测试，在你提交代码的那一瞬间，测试脚本会跑一遍所有功能：

* ❌ 登录失败。
* ❌ 购物车结算错误。

它阻止了你犯傻。

## Jest vs Vitest

* **Jest**：老牌王者，Facebook 出品。生态最丰富，但配置稍微有点慢。
* **Vitest**：新晋挑战者。基于 Vite。**极快**。完美兼容 Jest 的语法。

对于 **[Next.js](wiki:wiki_next_js)** 和 Vite 项目，现在的趋势是使用 **Vitest**。

## 怎么写？

你不需要测每一行代码。
**只测关键路径 (Happy Path)**。
比如："输入正确的用户名密码 -> 点击登录 -> 应该跳转到主页"。

## Vibe 建议

作为 Rookies，你可能觉得写业务代码都很吃力了，哪有时间写测试？
理解。
但请记住，**Unit Test (单元测试)** 是你睡好觉的保障。
至少，学会写几个简单的测试，感受一下看到 "✅ All Tests Passed" 时的多巴胺分泌。
