# 全能驾驶舱：Tools of Trade

> "工欲善其事，必先利其器。在 2025 年，你的'器'是外骨骼装甲。"

欢迎来到 **Vibe Coding 武器库**。
我们不应该随机选择工具。我们的工具链应该像特种部队的装备一样，经过精密搭配，每一件都有其战术目的。

---

## 1. 核心引擎：Cursor (AI IDE)

这不再是一个简单的选择题。VS Code 虽然好，但如果没有原生 AI 集成，它就像一把没有瞄准镜的枪。
**Cursor** 是目前的唯一真神。

### 深度配置指南 (Pro Config)

不要只用默认设置。

1. **模型选择**：
    * `CMD + K` (写代码)：设置为 **Claude 3.5 Sonnet** (或其后续版本)。这是目前写代码逻辑最强的模型。
    * `Chat` (问问题)：设置为 **Gemini 1.5 Pro** 或 **GPT-4o** (用于长上下文搜索)。
2. **规则系统 (.cursorrules)**：
    * 在项目根目录创建一个 `.cursorrules` 文件。这是给 AI 的"家规"。
    * **内容示例**：

        ```text
        - 始终使用 TypeScript。
        - 优先使用 Shadcn/UI 组件。
        - 永远不要修改我不让你修改的文件。
        - 回答要简洁，不要废话。
        ```

    * 有了它，AI 就不再是一个实习生，而是一个懂你习惯的老员工。
3. **Composer (Beta)**：
    * 按 `CMD + I` 打开 Composer。这是一个能同时修改多个文件的"上帝模式"。
    * 输入："把所有组件的颜色从蓝色改成紫色，并更新相关的 CSS 变量"。它可以一次性修改 20 个文件。

## 2. 特种部队：AI Agents

当任务太复杂，单一的 Chat 窗口无法解决时，你需要 **Agent (智能体)**。
它们能运行终端命令，能读写文件，能自我修正。

### 推荐武器：Cline (开源)

装在 VS Code / Cursor 里的插件。

* **工作流**：
    1. 给 Cline 一个模糊的任务："分析 `src/utils` 里的代码，找出重复的函数并重构它们。"
    2. Cline 会运行 `ls` 查看文件。
    3. Cline 会运行 `grep` 查找代码。
    4. Cline 会提出修改计划，并请求你批准。
    5. 你点击 "Approve"，它开始重写文件。
* **成本**：它需要你提供 API Key (OpenRouter / Anthropic)。但这几美分换来的是几个小时的人力节省。

---

## 3. 指挥中心：Warp (Terminal)

如果你还在用系统自带的终端，你就像在用记事本写代码。
**Warp** 是 21 世纪的终端。

* **AI Command Search**：忘了怎么解压 tar.gz？按 `#` 输入 "解压 tar 文件"，它自动给你命令。
* **Workflows**：你可以把常用的复杂命令保存为 Workflow，一键调用。
* **Block Select**：命令输出不再是一团乱码，而是一个个独立的块。你可以轻松复制某一次报错的日志。

---

## 4. 浏览器增强

* **Arc Browser**：它的侧边栏和分屏功能是为宽屏显示器设计的。配合 "Boost" 功能，你可以用 CSS 修改任何网站的外观（比如把 GitHub 变成暗黑模式）。
* **React Developer Tools**：必装。查看组件树，查看 State 变化。
* **VisBug**：一个 Chrome 插件，允许你像在 Figma 里一样直接修改网页元素（移动位置、改文字、改颜色）。对于微调 UI 极其有用。

## 总结

你的工具链不仅仅是软件，它是你的**外骨骼**。
Cursor 是你的机械臂，Warp 是你的HUD显示器，AI Agents 是你的无人机僚机。
熟悉它们，配置它们，直到它们成为你身体的一部分。
